# MS365 로그인 연동 계획

이 문서는 NextAuth.js(Auth.js)를 사용하여 UNI Customs 근태관리 대시보드에 Microsoft 365(Entra ID/Azure AD) 로그인을 연동하기 위한 계획을 설명합니다.

## 사용자 검토 필요 (User Review Required)

> [!IMPORTANT]
> **Microsoft Entra ID (Azure AD) 앱 등록 설정**
> 이 연동을 완료하려면, 나중에 관리자님께서 Microsoft 365 (Azure Portal)에 들어가서 애플리케이션을 등록하고 인증 자격 증명(키)을 발급받아 주셔야 합니다.
> 
> 다음 세 가지 정보가 필요합니다:
> 1.  `AZURE_AD_CLIENT_ID` (애플리케이션 ID)
> 2.  `AZURE_AD_CLIENT_SECRET` (클라이언트 암호 / Secret)
> 3.  `AZURE_AD_TENANT_ID` (디렉터리 / Tenant ID)
> 
> 코딩 작업이 모두 완료되면, Azure Portal에서 이 값들을 가져오는 방법을 제가 단계별로 안내해 드리겠습니다.

## 제안된 변경 사항 (Proposed Changes)

### 1. 패키지 설치
- `next-auth` 설치
- 보안을 위한 환경변수 설정

---

### 2. 환경 변수 파일 생성
#### [NEW] `.env.local`
- 민감한 Microsoft 자격 증명(ID, Secret 등)과 프론트엔드-백엔드 암호화를 위한 `NEXTAUTH_SECRET` 값을 안전하게 저장할 환경 변수 파일을 생성합니다.

---

### 3. 로그인 인증 API 라우터 구축
#### [NEW] `frontend/src/app/api/auth/[...nextauth]/route.ts`
- NextAuth에서 제공하는 기본 `AzureADProvider`를 사용하여 인증 핸들러(API)를 구축합니다.
- JWT(JSON Web Token) 세션 전략을 사용하고, 로그인한 사용자의 이름과 이메일 정보를 프론트엔드로 전달하는 콜백을 구성합니다.

---

### 4. 클라이언트 세션 연동 (프론트엔드)
#### [NEW] `frontend/src/components/Providers.tsx`
- 인증 상태(세션)를 관리하는 `SessionProvider`를 제공하는 클라이언트 컴포넌트를 생성합니다.
#### [MODIFY] `frontend/src/app/layout.tsx`
- 핵심 레이아웃 뼈대에 새로 만든 `<Providers>` 컴포넌트를 감싸서, 애플리케이션 어디서든(`Sidebar`, `Header` 등) 로그인 정보를 읽을 수 있도록 만듭니다.

---

### 5. 로그인 UI 화면 및 보안(라우트 타당성) 처리
#### [NEW] `frontend/src/app/login/page.tsx`
- UNI Customs 테마에 맞춘 깔끔한 디자인의 로그인 페이지를 생성합니다. 중앙에 "Microsoft 계정으로 로그인" 버튼을 배치합니다.
#### [NEW] `frontend/src/middleware.ts`
- Next.js 미들웨어를 생성하여, 강력한 보안을 적용합니다. 로그인하지 않은(인증되지 않은) 사용자가 주소창에 `/employees`나 `/attendance` 등을 직접 쳐서 들어와도 무조건 `/login` 페이지로 튕겨내도록(Redirect) 자동 보호합니다.
#### [MODIFY] `frontend/src/components/Header.tsx`
- 헤더 우측 상단의 사용자 프로필 메뉴를 수정하여, **실제로 MS365를 통해 로그인한 사람의 이름과 이메일**이 표시되도록 바꿉니다.
- 프로필 하단의 "Sign out(로그아웃)" 버튼이 실제로 작동하여 세션을 종료하도록 기능을 연결합니다.

## 검증 계획 (Verification Plan)

### 수동 검증 단계
1.  **접근 차단 확인:** 개발 서버를 켜고, 무작정 메인 화면(`/`)으로 접속을 시도했을 때, 안전하게 `/login` 화면으로 쫓겨나는지 확인합니다.
2.  **Microsoft 인증 흐름 연동:** "Sign in with Microsoft" 버튼을 클릭했을 때, 실제 마이크로소프트의 기본 로그인 팝업이나 화면이 제대로 뜨는지 검증합니다.
3.  **인증 성공 후 데이터 연동:** 회원가입/로그인 완료 시, 대시보드로 다시 잘 돌아오는지(Redirect) 확인하고 프로필 메뉴에 MS365 계정 정보가 정확하게 노출되는지 확인합니다.
4.  **로그아웃 흐름:** 로그아웃을 클릭하면 세션이 안전하게 지워지고 다시 로그인 화면으로 돌아가는지 확인합니다.
